Politica de Branches (GitFlow)
main Produccion — Solo recibe merges de release/* y hotfix/* Bloqueada
develop Integracion — Aqui convergen todas las features terminadas PR Requerido
feature/* Nuevas funcionalidades — Se crea desde develop, se mergea a develop Abierta
release/* Preparacion de release — Desde develop, se mergea a main y develop PR Requerido
hotfix/* Fixes urgentes en produccion — Desde main, se mergea a main y develop PR Requerido

Referencia Rapida de Comandos
Clonar repositorio git clone git@github.com:GoEcosystemDH/mi-repo.git
Crear feature branch git checkout develop && git pull && git checkout -b feature/TICKET-123-descripcion
Commit con convencion git commit -m "feat(modulo): descripcion del cambio"
Push y crear PR git push -u origin feature/TICKET-123-descripcion
Actualizar develop local git checkout develop && git pull origin develop
Despues del merge, limpiar git branch -d feature/TICKET-123-descripcion
