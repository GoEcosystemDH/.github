¿Que necesito hacer?

1 Si ya tenias el repo clonado desde GitLab, actualiza el remote:

Esto le dice a Git que ahora el codigo vive en GitHub. No pierdes nada local.

git remote set-url origin https://github.com/GoEcosystemDH/NOMBRE-REPO.git
2 Si es la primera vez, clona desde GitHub: 
git clone https://github.com/GoEcosystemDH/NOMBRE-REPO.git
3 Verifica que apunta al lugar correcto: 
git remote -v
# Deberia mostrar:
# origin  https://github.com/GoEcosystemDH/NOMBRE-REPO.git (fetch)
# origin  https://github.com/GoEcosystemDH/NOMBRE-REPO.git (push)
Reemplaza NOMBRE-REPO con el nombre que aparece en la columna "Repo GitHub" de la tabla.
Siguiente paso: una vez actualizado el remote, revisa la guia completa y el checklist para asegurarte de que todo funciona bien.

¿Que significan las columnas?

La tabla de abajo tiene cinco columnas. Esto es lo que significa cada una:

  • Repo GitLab — El nombre original que tenia el repositorio en GitLab.
  • Repo GitHub — El nuevo nombre en GitHub. Haz click en el enlace para ir directamente al repo.
  • Rama — La rama principal del proyecto (main, master, develop, etc.).
  • Pipeline — Si = el repo tenia un proceso automatico de compilacion o despliegue (CI/CD) en GitLab. No = no tenia pipeline.
  • CI/CD Migrado — Si = ese proceso automatico ya funciona en GitHub Actions. No = aun se esta migrando o no aplica.

  • Necesitas hablar con el equipo DevOps para verificar su estado.
